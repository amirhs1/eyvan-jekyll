--- layout: default --- {% comment %} Post Layout Purpose: Render long-form article pages used for projects, essays, research notes, and documentation posts. Structure: - Post page container - Optional cover image (full-width) - Article wrapper - Post header (full-width) - Optional tag list - Post title - Optional subtitle - Header row (metadata + share) - Post body content - 3-column content layout (desktop) - Left column (empty, layout symmetry) - Main column (prose content) - Right column (Table of Contents) - Single-column layout (mobile/tablet) - Prose content - TOC hidden by default - Mobile TOC toggle button (floating control) - Post navigation (full-width) - Related posts section (full-width) Layout: - Header and media span full container width - Content area uses a symmetric 3-column system: | left column | prose | right column (TOC) | - Prose width is constrained via o-prose (ch-based measure) - Left and right columns remain equal to preserve visual centering Table of Contents (TOC): - Desktop: - Rendered in the right column - Sticky within the post content boundary - Does not overlap header, navigation, or related posts - Mobile: - Hidden by default - Accessible via floating toggle button - Controlled via JavaScript (data-toc-toggle) - Generated via `_includes/toc.html` (Liquid-based, no plugins) Architecture: - Layout layer of the site - Uses ITCSS + BEM naming - Content layout handled by `c-post-layout` - Columns defined as: - c-post-layout__col - c-post-layout__col--left - c-post-layout__col--main - c-post-layout__col--right - Delegates reusable UI fragments to: - entry-meta.html - post-share.html - post-navigation.html - related-posts.html - toc.html Accessibility: - Uses semantic article and header elements - TOC uses
with aria-labelledby - Mobile TOC toggle uses aria-expanded and aria-controls - Decorative layout columns are hidden from assistive tech (aria-hidden) Data dependencies: - page.title - page.subtitle - page.image - page.image_alt - page.tags - page.share - page.toc - content - entry-meta.html dependencies - post-share.html dependencies - post-navigation.html dependencies - related-posts.html dependencies Notes: - This layout owns the overall page structure for individual posts - Reusable post metadata is rendered through entry-meta.html - Global spacing is owned by .l-site-main in default.html - Share actions can be disabled with: page.share: false - TOC can be enabled per post with: page.toc: true {% endcomment %}
