O-Site-Header Test

This page verifies the functionality of the .o-site-header object and its layout zones defined in the 4-objects Sass layer.

Development note: the helper styles on this page exist only to make the header layout boundaries easier to inspect visually.

Base mobile header layout

Expected behavior: On smaller screens, the brand area should appear on the left and the mobile controls area should appear on the far right because .o-site-header__mobile-controls uses margin-inline-start: auto. The desktop navigation and desktop actions zones should remain hidden.
