This page verifies the functionality of the .o-site-header object and its structural child zones defined in the 4-objects Sass layer.

Development note: the helper styles on this page exist only to make the layout boundaries easy to inspect.

Base object layout

Expected behavior: The header should render as a single horizontal flex row. The brand zone should appear first. On small screens, the mobile controls zone should be pushed to the far end by margin-inline-start: auto.

Expected behavior: On smaller screens, only the brand zone and mobile controls zone should be visible. The nav and actions zones should be hidden.

Layout breakpoint visibility

Expected behavior: At the layout breakpoint and above, the nav zone and actions zone should become visible, while the mobile controls zone should become hidden. The nav zone should expand and sit between the brand and actions zones.
