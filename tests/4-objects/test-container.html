O-Container Test

This page verifies the functionality of the .o-container object and the .o-container--full modifier defined in the 4-objects Sass layer.

Development note: the helper styles on this page exist only to make container boundaries easier to inspect visually.

Base container behavior

Expected behavior: The dashed box below should appear centered in the viewport with visible horizontal space on both sides. The content should not touch the browser edges because .o-container applies inline gutter padding.

Base test: This block is wrapped by .o-container.

Viewport edge separation

Expected behavior: The dashed test box should never touch the outer boundary of the viewport. The .o-container inline padding should keep content separated from the viewport edges at all screen sizes.

Edge test: This box should always remain inset from the viewport edges.

Container inside full-width section

Expected behavior: The gray background section should stretch across the full browser width, while the dashed content box inside it should remain centered and width-constrained. This verifies that .o-container works correctly inside a full-width layout section.

Containment test: The section background is full width, but this content stays constrained.

Max-width constraint

Expected behavior: On narrow screens, the container should expand naturally to fit the viewport minus its inline gutters. On wide screens, it should stop growing once it reaches its configured maximum width.

Max-width test: Resize the browser to a very wide viewport. This box should eventually stop expanding horizontally.

Full-width modifier

Expected behavior: The dashed box below should span the full available width of its parent while still keeping inline gutter padding. Unlike the default container, it should not stop growing at the normal max-width.

Full modifier test: This wrapper uses .o-container.o-container--full.

Responsive inline gutters

Expected behavior: At smaller viewport widths, the container should use the smaller inline gutter value. At the configured wide breakpoint, the left and right padding should become visibly larger.

Gutter test: Watch the space between this box and the viewport edge as you resize. The inline padding should increase at the wide breakpoint.

Tip: Use browser DevTools to inspect .o-container and confirm the computed padding-inline value before and after the wide breakpoint.
