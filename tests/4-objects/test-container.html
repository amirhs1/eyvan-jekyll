This page verifies the functionality of the .o-container object and the .o-container--full modifier defined in the 4-objects Sass layer.

Development note: helper surfaces on this page come from the shared test helpers in 7-trumps/_tests.scss.

Base container behavior

Expected behavior: The test box below should appear centered in the viewport with visible horizontal space on both sides. The content should not touch the browser edges because .o-container applies inline gutter padding.

Base test: This block is wrapped by .o-container.

Viewport edge separation

Expected behavior: The test box should never touch the outer viewport boundary. The container’s inline padding should keep content separated from the viewport edges at all screen sizes.

Edge test: This box should always remain inset from the viewport edges.

Container inside full-width section

Expected behavior: The outer helper section should stretch across the full browser width, while the content box inside it should remain centered and width-constrained. This verifies that .o-container works correctly inside a full-width layout section.

Containment test: The section background is full width, but this content stays constrained.

Max-width constraint

Expected behavior: On narrow screens, the container should expand naturally to fit the viewport minus its inline gutters. On wide screens, it should stop growing once it reaches its configured maximum width.

Max-width test: Resize the browser to a very wide viewport. This box should eventually stop expanding horizontally.

Full-width modifier

Expected behavior: The box below should span the full available width of its parent while still keeping inline gutter padding. Unlike the default container, it should not stop growing at the normal max-width.

Full modifier test: This wrapper uses .o-container.o-container--full.

Responsive inline gutters

Expected behavior: The container uses fluid inline gutters by default, and those gutters become larger at the wide breakpoint.

Gutter test: Watch the space between this box and the viewport edge as you resize. The inline padding should become larger on wide screens.

Tip: inspect .o-container in DevTools and compare the computed padding-inline before and after the wide breakpoint.
