Object Prose Test

This page verifies the functionality of the prose object classes defined in the 4-objects Sass layer.

Scoped readable measure

Expected behavior: The prose block below should not stretch to the full width of its wider parent container. Instead, .o-prose.o-prose--measure-base should apply a readable text measure and center the content with automatic inline margins.

Readable line length

This paragraph is intentionally placed inside a wider test box so that the prose object can demonstrate its own measure. The text column should remain comfortably readable rather than spanning the full available width of the parent container.

The left and right edges of the prose content should appear centered within the wider test frame, showing that the object owns readable line length independently of page-shell containment.

Scope containment outside the prose object

What to look for before comparing:

  • The .o-prose example should form a more readable long-form text column instead of stretching too comfortably across the available width.
  • Spacing between the heading, paragraphs, list, and final paragraph should feel more deliberate and article-like in the .o-prose block.
  • List indentation, nested-list spacing, and inline-link treatment should be controlled by the prose object only inside the .o-prose block.
  • The non-prose example should render as ordinary page content inside a plain container, without the scoped long-form behavior of .o-prose.
