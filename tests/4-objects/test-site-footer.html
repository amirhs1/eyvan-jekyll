This page verifies the functionality of the .o-site-footer object and its structural child zones defined in the 4-objects Sass layer.

Development note: the helper styles on this page exist only to make the layout boundaries easy to inspect.

Base object layout

Expected behavior: The footer should render as a vertical flex stack on smaller screens. The brand zone should appear first, the meta zone second, and the actions zone third. All zones should remain centered horizontally.

Mobile-first stacking

Expected behavior: On smaller screens, the footer regions should stack vertically because the object uses flex-direction: column. The brand and actions zones should remain horizontally centered, and the meta zone should also stay centered.

Meta centered on small screens
Actions centered on small screens

Layout breakpoint shift

Expected behavior: At the layout breakpoint and above, the footer should switch to a horizontal row. The brand zone should align to the start, the meta zone should remain centered, and the actions zone should align to the end.

Meta stays centered at layout breakpoint
Actions aligned to end at layout breakpoint

Equal region distribution

Expected behavior: At the layout breakpoint and above, the three footer zones should behave like equal-width flex regions because the brand, meta, and actions zones each use flex: 1 1 0%.
