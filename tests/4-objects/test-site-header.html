Object Site Header Test

This page verifies the functionality of the .o-site-header object and its structural child zones defined in the 4-objects Sass layer.

Development note: helper styles on this page exist only to make layout boundaries easier to inspect.

Base object layout

Expected behavior: The header should render as a single horizontal flex row. The brand zone should appear first. On small screens, the mobile controls zone should be pushed to the far end by margin-inline-start: auto.

Layout breakpoint visibility

Expected behavior: At the layout breakpoint and above, the nav, theme, and actions zones should become visible, while the mobile controls zone should become hidden. The nav zone should expand and sit between the brand and the desktop controls.

Long content stress test

Expected behavior: This test should stress the zone that is actually flexible in the header object. At the layout breakpoint and above, the nav zone should absorb long content and wrap without causing obvious horizontal overflow, because it is the middle flexible region using flex: 1 1 auto and min-inline-size: 0. The brand, theme, and actions zones remain content-sized edge regions.
