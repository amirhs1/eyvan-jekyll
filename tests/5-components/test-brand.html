This page verifies the functionality of the brand classes defined in the 5-components Sass layer.

Base brand structure

Expected behavior: The brand should render as a single clickable inline-flex unit with a logo on the left and a two-line text stack on the right. The text stack should show the name and role, and the entire block should behave like one homepage link.

John Doe Computational Social Scientist

Long-text truncation behavior

Expected behavior: When the brand text becomes too long, both .c-brand__name and .c-brand__role should remain on a single line and truncate cleanly with ellipsis rather than wrapping. This test uses a temporary helper class so the brand fills the narrow test box and the truncation can be verified reliably.

John Doe With a Very Long Professional Display Name Interdisciplinary Researcher in Computational Social Science and Digital Humanities

Narrow-width collapse behavior

Expected behavior: On tighter viewports, the component should collapse from logo + text to logo only. The text stack should visually disappear by reducing its maximum inline size and opacity.

John Doe Computational Social Scientist

Resize the browser window below the medium breakpoint to confirm that the text stack collapses away and the logo shrinks to its collapsed size.

Header shrunk-state behavior

Expected behavior: When an ancestor has .c-site-header.is-shrunk, the brand should also collapse to its compact logo-first treatment even if the viewport is otherwise wide enough for the full text stack.

No-role variant

Expected behavior: If the secondary role line is absent, the component should still remain visually aligned and usable as a single homepage link.

John Doe

Logo-only stress case

Expected behavior: The component should remain valid and clickable even in a logo-only markup scenario. This helps verify that the logo behaves as a stable visual unit independent of the text stack.

Tip: Test both keyboard focus and browser resizing. The focus ring should appear on the whole brand link, while hover and focus should keep the text colors stable.
