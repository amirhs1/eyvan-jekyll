This page verifies the functionality of the .c-button component and its modifiers defined in the 5-components Sass layer.

Development note: the helper styles on this page exist only to make button boundaries, spacing, and wrapping behavior easier to inspect.

Base button

Expected behavior: The base button should render as an inline-flex control with centered content, visible padding, rounded corners, no text underline, and stable alignment whether it is rendered as a link or a button element.

Base button as link

Primary and secondary variants

Expected behavior: The primary and secondary variants should look visually distinct while preserving the same button structure, spacing, and typography. Hover and focus-visible states should change the visual treatment without breaking layout.

Primary action Secondary action

Size modifiers

Expected behavior: The small and large modifiers should change the button height, padding, and font size while preserving alignment, border radius, and variant styling.

Secondary sizes: Small secondary Large secondary

Icon composition

Expected behavior: Buttons using .c-button--with-icon should align the icon circle and label on one line with consistent spacing. The icon circle should remain circular and should not stretch.

↓ Secondary with icon → Large primary with icon

Icon button with longer label

Expected behavior: The icon circle should stay fixed in size, and the label should remain aligned with it. This checks that .c-button__label and the icon elements compose cleanly together.

Disabled state

Expected behavior: Disabled buttons should appear visually muted and should not behave like interactive controls. This verifies both the native :disabled state and the [aria-disabled="true"] pattern.

Small-screen behavior

Expected behavior: On very narrow screens, buttons should still remain usable and not collapse awkwardly. Above the component’s small breakpoint, the button can expand to its content width more confidently. Resize the viewport to inspect this.

White-space stress test

Expected behavior: Because the button uses white-space: nowrap, a very long label should stay on one line. This test is useful for spotting overflow risk and deciding whether your real content stays within sensible button lengths.
