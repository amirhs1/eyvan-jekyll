This page verifies the functionality of the navigation classes defined in the 5-components Sass layer.

Base navigation structure

Expected behavior: The navigation list below should render as a horizontal flex row with aligned items, consistent spacing between links, and no default list bullets because .c-nav__list is used together with .u-list-reset.

  • Home
  • Projects
  • About
  • Contact

Desktop navigation variant

Expected behavior: The desktop variant uses .c-nav--desktop. Below the layout breakpoint it should be hidden. At and above the layout breakpoint it should become visible.

  • Home
  • Projects
  • Writing
  • Research

Resize the browser width to confirm the desktop navigation visibility rule.

Active and current-page states

Expected behavior: Links with .is-active or aria-current="page" should use the active color. In the default or desktop-style navigation, they should also show a small circular marker near the bottom.

  • Overview
  • Projects
  • Research
  • Contact

Desktop list wrapping behavior

Expected behavior: In the desktop variant, the list may wrap onto multiple rows because .c-nav--desktop .c-nav__list allows wrapping. However, each individual link should still stay on a single line because .c-nav__link uses white-space: nowrap.

  • Public Writing
  • Selected Projects
  • Research Notes
  • Contact

Mobile navigation variant

Expected behavior: The mobile variant should stack links vertically, stretch items to the available width, align text to the left, increase link size, and add a bottom border to each link. Its active link should not render the circular marker used by the default or desktop style.

  • Home
  • Projects
  • Publications
  • Contact

Mobile long-label wrapping behavior

Expected behavior: In the mobile variant, long labels should be able to wrap because the mobile modifier changes link whitespace behavior from nowrap to normal.

  • Very Long Navigation Label Example That Should Wrap in the Mobile Menu
  • Another Long Editorial Navigation Label for Responsive Wrapping

External-link behavior

Expected behavior: The external link below should look visually like the other navigation links. In your real include, external links also receive target="_blank" and rel="noopener noreferrer".

  • Projects
  • External CV

Header mobile toggle

Expected behavior: The toggle button should render as a circular icon button. On hover it should change color and background. The version with aria-expanded="true" should show the expanded state styling, including the raised-surface background and border. At and above the layout breakpoint, the toggle should become hidden.

Collapsed state: aria-expanded="false"

Expanded state: aria-expanded="true"

Tip: Resize the browser window to confirm that .c-nav--desktop and .c-nav-toggle switch visibility at the layout breakpoint.
