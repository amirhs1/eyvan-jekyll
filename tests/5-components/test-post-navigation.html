This page verifies the functionality of the post navigation classes defined in the 5-components Sass layer.

Base two-card navigation

Expected behavior: The root component should render two cards side by side when space allows. The previous card should place media first and content second. The next card should place content first and media second. Labels, titles, spacing, and alignment should feel consistent across both cards.

Previous-only state

Next-only state

No-navigation state

Expected behavior: In the real include, nothing should render when neither previous nor next exists. This test shows an intentionally empty frame so you can confirm that the correct behavior for that state is the absence of the component.

Intentionally empty: no .c-post-nav markup should be rendered when no previous or next post exists.
