This page verifies the functionality of the .c-theme-toggle component and its variants defined in the 5-components Sass layer.

Development note: the helper styles on this page exist only to make component boundaries and state behavior easier to inspect.

Base component rendering

Expected behavior: The base theme toggle should render as a compact circular button. It should contain the icon group and no visible text label in the default desktop-style form.

Compact desktop variant

Expected behavior: The default non-mobile version should keep a fixed square footprint. It should center its icon group and remain visually compact for header usage.

Desktop example
Icon-only header-style toggle.

Mobile labeled variant

Expected behavior: The .c-theme-toggle--mobile modifier should expand the control to fill the available inline space on smaller screens, align its contents to the start, and show the visible text label.

Light theme icon state

Expected behavior: In light theme, the moon icon should be visible because the action is to switch to dark theme. The sun icon should be visually hidden.

Dark theme icon state

Expected behavior: In dark theme, the sun icon should be visible because the action is to switch to light theme. The moon icon should be visually hidden.

Disabled states

Expected behavior: Disabled versions should appear reduced in opacity, should not suggest interactivity, and should not apply the active press transform.

Width and alignment behavior

Expected behavior: The mobile modifier should stretch to fill a constrained shell on smaller screens, while the default version should remain compact and content-sized.

Long label stress test

Expected behavior: This test stresses the labeled mobile variant. The control should remain usable and visually stable even with a long visible label. The button should still align its icon and text cleanly and stay inside the available width.

Resize the viewport and confirm that the mobile variant remains inside its container. Also verify that the default desktop variant stays compact while the mobile modifier behaves like a wider labeled control.
