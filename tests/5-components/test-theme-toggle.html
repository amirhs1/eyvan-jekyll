This page verifies the .c-theme-toggle component defined in the 5-components layer.

Base component rendering

Expected behavior: The base toggle should render as a compact circular button with an icon group and no visible label.

Compact desktop variant

Expected behavior: The default non-mobile version should keep a fixed square footprint and centered icon group.

Desktop example
Icon-only header-style toggle.

Mobile labeled variant

Expected behavior: The mobile modifier should fill the available inline space, align content to the start, and show the visible label.

Light theme icon state

Expected behavior: In light theme, the moon icon should be visible because the action is to switch to dark theme.

Dark theme icon state

Expected behavior: In dark theme, the sun icon should be visible because the action is to switch to light theme.

Disabled states

Expected behavior: Disabled versions should appear lower-emphasis, should not suggest interactivity, and should not use the active press transform.

Width and alignment behavior

Expected behavior: The mobile modifier should stretch to fill a constrained shell, while the default version remains compact.

Long label stress test

Expected behavior: The labeled mobile variant should remain usable and visually stable with a longer visible label.
