O-Grid Test

This page verifies the functionality of the .o-grid object and the .o-grid__item span and push utility classes defined in the 4-objects Sass layer.

Development note: the helper styles on this page exist only to make grid boundaries easier to inspect visually.

Base grid row behavior

Expected behavior: The two boxes below should sit on separate rows because each .o-grid__item defaults to inline-size: 100% unless a span utility changes it. The row should also apply horizontal negative margins to balance each item's inline padding.

Item 2 Default .o-grid__item

Equal column spans

Expected behavior: The three boxes below should sit on the same row, each taking one third of the available width because they use .o-grid__item--span-4 in a 12-column grid.

Mixed column spans

Expected behavior: The first row should contain a 3-column, 6-column, and 3-column layout. This verifies that different span utilities can be composed in the same row and still total 12 columns.

Span 3 --span-3
Span 6 --span-6
Span 3 --span-3

Base push utilities

Expected behavior: The first box below should begin after a 2-column offset because it uses .o-grid__item--push-2. The second box should wrap naturally to the next available row if there is not enough remaining space.

Push 2 + Span 4 --push-2 --span-4
Span 4 --span-4

Tip: Inspect the pushed item in DevTools and confirm that margin-inline-start is a percentage based on the configured 12-column grid.

Tablet span utilities

Expected behavior: On small screens, both items should appear full width because only tablet utilities are applied. At the tablet breakpoint, they should become two equal columns using .o-grid__item--span-tablet-6.
